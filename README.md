# 供应商管理系统 (Supply Chain Master)

跨境电商供应链开发的本地供应商信息管理工具。从选品决策到供应商维护的全流程管理，支持 1688 类目体系、新品跟进、报价追踪和产品对比。

## 快速开始

### 环境要求
- Python 3.9+
- Flask（`pip install flask`）
- openpyxl（`pip install openpyxl`，Excel 导入功能需要）

### 启动

```bash
cd workspace
pip install flask openpyxl
python app.py
# 浏览器打开 → http://localhost:5000
```

首次启动自动创建数据库并预置 36 个一级 / 280+ 个二级类目（基于 1688 平台）。

### 表结构详解

#### 1. categories — 1688 类目表

预置 36 个一级类目、280+ 二级类目，基于 1688 平台行业市场分类。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| level1 | TEXT | 一级类目 | 玩具 |
| level2 | TEXT | 二级类目 | 积木 |
| sort_order | INTEGER | 排序权重（越小越前） | 12 |

**一级类目排序优先级：**
- **高优先级（sort 1-13）：** 流行配饰、家具、礼品工艺品、家居用品、家电、灯具照明、箱包、办公文教、橡胶塑料、运动娱乐、五金工具、玩具、汽车配件
- **其他（sort 14-36）：** 农业、服装、美容与个人护理、商务服务、化工、建筑、消费电子、电气设备、电子元器件、能源、环保、定制加工、食品饮料、医药保健、机械、矿物冶金、包装印刷、促销品、安全防护、服务设备、鞋类配件、纺织皮革、钟表珠宝

#### 2. suppliers — 供应商表

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| name | TEXT | 公司全名（必填） | XX玩具有限公司 |
| short_name | TEXT | 简称（用于搜索匹配） | XX玩具 |
| address | TEXT | 地址 | 浙江省义乌市... |
| contact_person | TEXT | 联系人 | 张经理 |
| contact_info | TEXT | 联系方式 | 138xxxx |
| notes | TEXT | 备注 | 合作3年，交期稳定 |
| created_at | DATETIME | 创建时间 | 2025-01-20 10:00:00 |
| updated_at | DATETIME | 最后修改时间 | 2025-06-04 15:30:00 |

#### 3. supplier_categories — 供应商-分类关联表

实现多对多关系，附带三级类目。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| supplier_id | FK | 指向 suppliers.id | 1 |
| category_id | FK | 指向 categories.id | 12（玩具→积木） |
| level3 | TEXT | 三级类目（可选，仅供查看） | 大颗粒系列 |

> 一个供应商可属于多个分类（如积木+模型），一个分类下也可有多个供应商。
> 三级类目不参与筛选，仅在列表和详情中展示供参考。

#### 4. products — 产品表

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| erp_sku | TEXT PK | ERP系统SKU，为空则自动生成 | TOY-2401-001 |
| supplier_id | FK | 所属供应商 | 1 |
| name | TEXT | 产品全称（必填） | 城市系列消防局积木 |
| category_id | FK | 关联二级类目 | 12（玩具→积木） |
| product_url | TEXT | 1688 商品链接 | https://detail.1688.com/... |
| package_type | TEXT | 单套包装方式 | 彩盒 |
| package_size | TEXT | 单套包装尺寸 | 35×25×8 |
| package_weight | TEXT | 单套重量 | 580g |
| carton_size | TEXT | 外箱尺寸 | 72×52×42 |
| carton_quantity | INTEGER | 每箱套数 | 24 |
| carton_weight | TEXT | 外箱重量 | 14.5kg |
| is_new | BOOLEAN | 是否新品 | 1 |
| new_product_date | DATE | 新品上架日期 | 2025-01-10 |
| notes | TEXT | 备注 | 订单量≥1000可议价 |
| created_at | DATETIME | 创建时间 | |
| updated_at | DATETIME | 最后修改时间 | |

#### 5. price_history — 报价历史表

一个产品可以有多条报价记录，完整追踪价格变动。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| erp_sku | FK | 关联产品 | TOY-2401-001 |
| price | REAL | 报价金额（元） | 25.80 |
| price_date | DATE | 报价日期 | 2025-01-15 |
| source | TEXT | 来源（报价单号等） | 1月报价单 |
| notes | TEXT | 备注 | 满1000可议 |

> 产品详情页展示全部报价历史，最新报价在列表页显示。系统自动计算每次变动的涨跌金额（▲+1.2 / ▼-2.0），并生成简易价格趋势柱状图。

#### 6. product_images — 产品图片表

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| erp_sku | FK | 关联产品 | TOY-2401-001 |
| filename | TEXT | 图片文件名 | 消防局_正面.jpg |
| sort_order | INTEGER | 排列顺序 | 1 |
| uploaded_at | DATETIME | 上传时间 | |

> 文件中存储在 `uploads/{SKU}/{filename}`，支持 jpg/png/webp/gif/bmp 格式，可批量上传。

#### 7. follow_ups — 跟进记录表

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| supplier_id | FK | 关联供应商 | 1 |
| erp_sku | FK | 关联产品SKU（可选） | TOY-2401-001 |
| follow_date | DATE | 跟进日期 | 2025-01-20 |
| content | TEXT | 跟进内容 | 问了新品情况，说下月初出 |
| follow_type | TEXT | 类型（枚举） | 新品问询 |
| is_replied | BOOLEAN | 是否已回复 | 0=待回复, 1=已回复 |
| next_follow_date | DATE | 下次跟进提醒日期 | 2025-02-01 |
| created_at | DATETIME | 录入时间 | |

> 跟进类型包括：报价咨询、新品问询、常规维护、其他。
> 跟进记录可关联具体产品 SKU，也可不关联（通用沟通）。

#### 8. quotation_files — 报价表文件

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | INTEGER PK | 自动编号 | 1 |
| supplier_id | FK | 关联供应商 | 4 |
| display_name | TEXT | 自定义名称 | 1月消防局积木报价 |
| filename | TEXT | 存储文件名（含时间戳） | 20250115_143022_XX玩具报价.xlsx |
| original_name | TEXT | 上传前原始名 | XX玩具报价表.xlsx |
| notes | TEXT | 备注 | 含5款新品报价 |
| uploaded_at | DATETIME | 上传时间 | |

> 文件存储在 `quotations/` 平层目录，命名格式 `{供应商名}_{自定义名}_{日期}.{ext}`。支持上传任意格式文件（Excel/PDF/图片），点击 📂 打开文件夹直接弹出 Windows 资源管理器查看所有文件，拔 U 盘即可跨电脑迁移。

### 核心业务 SQL 查询示例

**筛选某二级分类下的产品，按价格排序：**
```sql
SELECT p.name, s.name, ph.price
FROM products p
JOIN suppliers s ON p.supplier_id = s.id
LEFT JOIN price_history ph ON p.erp_sku = ph.erp_sku
WHERE p.category_id = 12
ORDER BY ph.price ASC
```

**统计本月新品数：**
```sql
SELECT COUNT(*) FROM products
WHERE is_new = 1 AND new_product_date >= '2025-06-01'
```

**待跟进记录数（已到提醒日期但未回复）：**
```sql
SELECT COUNT(*) FROM follow_ups
WHERE is_replied = 0 AND next_follow_date <= '2025-06-04'
```

## 功能模块

### 仪表盘首页
- **统计卡片：** 供应商总数、本月新品数、待跟进数
- **最近报价变动：** 列表展示最近有价格变动的产品及涨跌金额
- **快捷入口：** 新增供应商、新增产品、新品动态、产品列表

### 供应商管理
- **列表：** 分页展示所有供应商，支持级联筛选（一级→二级）、关键词搜索（名称/简称/联系人）
- **新增/编辑：** 一级类目下拉选择 → 二级复选框勾选 → 三级类目自定义填写，已选分类实时显示在摘要区，支持 ✕ 一键移除
- **详情：** 展示供应商基本信息 + 关联分类（含三级） + 关联产品列表 + 跟进记录

### 产品报价管理
- **列表：** 筛选（级联分类/供应商/仅新品） + 排序（价格低到高/高到低/最新报价） + 对比勾选
- **新增/编辑：** 供应商搜索过滤 + 级联分类 + 包装数据 + 多图上传 + 首次报价
- **详情：** 基本信息 + 包装数据 + 图片轮播 + 报价历史（含价格趋势柱状图） + 相关跟进记录

### Excel 批量导入
- 下载预填模板（含表头 + 示例数据），按 16 列规范填好后上传
- 自动匹配供应商名称（支持简称）和分类（"一级→二级"格式）
- SKU 为空自动生成，重复自动跳过
- 导入后显示成功/失败详情

### 产品对比
- 在产品列表勾选 2-5 个产品，进入横向对比视图
- 对比字段：报价、包装数据（类型/尺寸/重量）、外箱数据、1688 链接、备注
- 最低报价列自动高亮绿色并标记"💰 最低价"

### 新品动态台
- 按供应商分组展示标记为新品的产品
- 每个新品标注跟进状态（已跟进/待跟进）
- 显示该供应商下次跟进提醒日期

### 跟进记录
- 支持按类型分类（报价咨询/新品问询/常规维护/其他）
- 可关联具体产品 SKU
- 设定下次跟进日期自动提醒

## 部署与迁移

### 跨电脑迁移（U盘方式）

```
U盘 /
├── 项目文件夹/              ← 整个 workspace 目录
│   ├── app.py
│   ├── templates/
│   ├── static/
│   ├── requirement.txt
│   ├── supplier.db          ← 你的数据
│   └── uploads/             ← 你的图片
```

目标电脑需先安装 Python 和 Flask。之后双击 `python app.py` 启动。

### 数据备份

只需要备份两个东西：
- `supplier.db` — 所有结构化数据
- `uploads/` — 产品图片

## 打包为 exe

如果公司电脑不能安装 Python，可用 PyInstaller 打包：

```bash
pip install pyinstaller
pyinstaller --onefile --add-data "templates;templates" --add-data "static;static" app.py
```

生成 `dist/app.exe`。

> **注意：** 更新代码后需重新打包覆盖旧 exe。如果新增了数据库字段，需要删除旧 `supplier.db` 让系统重建。
